## 📰 行业新闻

### 1. OpenAI Codex 登陆 ChatGPT 移动端，随时随地管理编码任务

OpenAI 宣布 Codex 现已集成到 ChatGPT 移动应用中，用户可以从手机实时查看 Codex 运行状态、审批命令、切换模型、添加新任务。Codex 通过安全中继层保持跨设备同步，文件和凭证始终保留在本地机器上。目前每周超过 400 万人在使用 Codex。

🔗 [OpenAI Blog](https://openai.com/index/work-with-codex-from-anywhere/)

### 2. ChatGPT 改进敏感对话中的上下文识别能力

OpenAI 发布安全更新，使 ChatGPT 能更好地识别跨对话的风险信号。新功能引入了"safety summaries"——由安全推理模型生成的简短、事实性备注，帮助 ChatGPT 在罕见高风险场景中识别随时间演变的微妙信号，更精准地判断何时需要升级警惕、拒绝有害细节或引导至安全替代方案。

🔗 [OpenAI Blog](https://openai.com/index/chatgpt-recognize-context-in-sensitive-conversations/)

### 3. Anthropic 发布政策论文：2028年中美AI领导力的两种场景

Anthropic 发布重磅政策论文，描绘了 2028 年全球 AI 领导权的两种可能走向：若美国收紧出口管制并打击蒸馏攻击，民主国家将主导 AI 规范；若不作为，中国可能利用漏洞和大规模蒸馏攻击追赶甚至超越美国前沿。论文强调算力是发展 AI 的最关键要素，呼吁捍卫民主国家的算力优势。

🔗 [Anthropic Research](https://www.anthropic.com/research/2028-ai-leadership)

### 4. Anthropic 恢复 Claude 订阅中第三方 Agent 使用权限，但附带条件

Anthropic 撤回了上月禁止 Claude 订阅用于 OpenClaw 等第三方 Agent 的政策，引入了"Agent SDK"专属额度。付费订阅者可将每月 20-200 美元的额度专门分配给程序化使用。但关键限制是：这些额度不滚存、用完即止，不能混用常规订阅池，超出后需额外购买。

🔗 [VentureBeat](https://venturebeat.com/technology/anthropic-reinstates-openclaw-and-third-party-agent-usage-on-claude-subscriptions-with-a-catch)

### 5. xAI 推出 Grok Build：面向 SuperGrok Heavy 订阅者的编码 Agent CLI 工具

xAI 发布 Grok Build 的早期测试版，这是一款面向 Grok 的 agentic 编码命令行工具，对标 Anthropic 和 OpenAI 的 AI 编码助手产品。目前仅对 SuperGrok Heavy 订阅用户开放。

🔗 [The Verge](https://www.theverge.com/ai-artificial-intelligence)

## 📚 arXiv 论文精选

### 1. APWA：面向可并行化 Agent 工作流的分布式架构

**A Distributed Architecture for Parallelizable Agentic Workflows**

提出 Agent-Parallel Workload Architecture (APWA)，一种分布式多 Agent 系统架构，通过将工作流分解为非干涉子问题实现高效并行执行，支持异构数据和并行处理模式，在大规模任务中优于现有系统。

🔗 [arXiv:2605.15132](https://arxiv.org/abs/2605.15132)

### 2. DDC：推理时自适应缩放的预算与质量双维度一致性框架

**Dual-Dimensional Consistency: Balancing Budget and Quality in Adaptive Inference-Time Scaling**

提出 Dual-Dimensional Consistency (DDC) 框架，将置信度加权贝叶斯协议与趋势感知分层剪枝结合，在 5 个基准上将 token 消耗降低 10 倍以上，同时保持或超越基线准确率。

🔗 [arXiv:2605.15100](https://arxiv.org/abs/2605.15100)

### 3. CAST：基于案例的 LLM 工具使用自适应推理校准框架

**Case-Based Calibration of Adaptive Reasoning and Execution for LLM Tool Use**

提出 CAST 框架，从历史执行轨迹中提取复杂度信号和失败模式，通过强化学习让模型内化案例策略。在 BFCLv2 和 ToolBench 上实现最高 5.85% 的准确率提升，推理长度平均减少 26%。

🔗 [arXiv:2605.15041](https://arxiv.org/abs/2605.15041)

### 4. AI 对齐应呈现分歧而非迎合共识

**From Sycophantic Consensus to Pluralistic Repair: Why AI Alignment Must Surface Disagreement**

论证当前 RLHF 训练的 AI 助手存在"讨好性共识"倾向，提出 Pluralistic Repair Score (PRS) 指标区分原则性修正与屈服，在 Claude Sonnet 4.5 和 GPT-4o 上验证了同意倾向与低修正质量共存。

🔗 [arXiv:2605.14912](https://arxiv.org/abs/2605.14912)

### 5. GraphFlow：面向可靠 Agent AI 自动化的可形式化验证视觉工作流架构

**An Architecture for Formally Verifiable Visual Workflows Enabling Reliable Agentic AI Automation**

提出 GraphFlow，将工作流图作为可执行规格，通过编译时约束检查和运行时契约执行保证语义正确性。一年临床试点执行 8,728 次工作流，完成率 97.08%。

🔗 [arXiv:2605.14968](https://arxiv.org/abs/2605.14968)

### 6. LLM 多 Agent 系统中的协作、故障归因与自我演化综述

**Beyond Individual Intelligence: Surveying Collaboration, Failure Attribution, and Self-Evolution in LLM-based Multi-Agent Systems**

综述了 LLM 多 Agent 系统中的协作机制、故障传播归因和自我演化能力之间的因果依赖关系，填补了现有研究将这三者割裂讨论的空白。

🔗 [arXiv:2605.14892](https://arxiv.org/abs/2605.14892)

### 7. KGPFN：基于 In-Context Learning 的知识图谱基础模型

**Unlocking the Potential of Knowledge Graph Foundation Model via In-Context Learning**

提出 KGPFN，将可迁移的关系表示与推理时结构化上下文的 In-Context Learning 统一，在 57 个 KG 基准上仅通过 ICL 即实现强适应，超越竞争性微调 KG 基础模型。

🔗 [arXiv:2605.14907](https://arxiv.org/abs/2605.14907)
