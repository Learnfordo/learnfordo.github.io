## 📰 行业新闻

### 1. 🚀 DeepSeek-V4 首日开源支持：SGLang + Miles 双引擎就位

LMSYS 宣布推出 DeepSeek-V4 的首日全面支持：SGLang 推理引擎和 Miles RL 训练引擎均已上线，成为首个在发布当天即支持 DeepSeek-V4 的开源技术栈。V4 模型（1.6T Pro / 284B Flash）引入了混合稀疏注意力（Hybrid Sparse-Attention）、流形约束超连接（mHC）和 FP4 专家权重三项关键架构创新，支持从 4K 到 1M 上下文窗口的推理吞吐保持稳定。

🔗 [LMSYS Blog](https://www.lmsys.org/blog/2026-04-25-deepseek-v4/)

### 2. 🧮 ChatGPT 业余爱好者破解 60 年未解 Erdős 难题

Scientific American 报道，一位非数学专业的业余爱好者使用 ChatGPT（"vibe math"方式）成功解决了一个已有 60 年历史的未解 Erdős 问题。该问题来自 Erdős Problems 网站，由著名数学家 Paul Erdős 提出。这一突破展示了 LLM 在辅助数学发现方面的潜力。

🔗 [Scientific American](https://www.scientificamerican.com/article/amateur-armed-with-chatgpt-vibe-maths-a-60-year-old-problem/) | [HN 讨论](https://news.ycombinator.com/item?id=47903126)

### 3. 🖥️ 用 AI 编程助手复活被你放弃的旧项目

🔗 [Matthew Brunelle Blog](https://blog.matthewbrunelle.com/its-ok-to-use-coding-assistance-tools-to-revive-the-projects-you-never-were-going-to-finish/) | [HN 讨论](https://news.ycombinator.com/item?id=47902525)

### 4. 🎮 AgentBox：在任意沙箱中运行 Claude Code / Codex / OpenCode 的 SDK

TwillAI 开源了 AgentBox SDK——一个可以在 Docker、E2B、Modal 等沙箱环境中运行 Claude Code、Codex、OpenCode 等编码 Agent 的统一接口。核心设计是通过启动 Agent 的原生服务器（JSON-RPC / HTTP-SSE / WebSocket）实现交互式会话驱动，而非简单的非交互式调用。

🔗 [GitHub/TwillAI](https://github.com/TwillAI/agentbox-sdk) | [HN 讨论](https://news.ycombinator.com/item?id=47876788)

### 5. 🏢 Cohorte 开源六库 AI Agent 治理栈

Cohorte 团队开源了面向企业 AI Agent 部署的完整治理栈，包含 6 个 Python 库：TrustGate（可靠性认证）、Guardrails（声明式 YAML 策略引擎）、Context Router（上下文路由）、Agent Monitor（可观测性）、Agent Auth（身份管理）等，全部基于 Apache 2.0 协议。

🔗 [Cohorte GitHub](https://github.com/Cohorte-ai) | [HN 讨论](https://news.ycombinator.com/item?id=47860859)

## 🔬 arXiv 最新论文

### 7. 🕸️ Mango：基于全局视图优化的多 Agent Web 导航

被 ACL 2026 接收。提出多 Agent Web 导航方法，将 URL 选择建模为多臂老虎机问题，使用 Thompson Sampling 自适应分配导航预算，并引入情景记忆组件存储历史导航经验。在 WebVoyager 上成功率 63.6%（GPT-5-mini），超过基线 7.3%；在 WebWalkerQA 上达 52.5%，超过基线 26.8%。

🔗 [arXiv:2604.18779](https://arxiv.org/abs/2604.18779) | [GitHub](https://github.com/VichyTong/Mango)

### 8. 🧠 CoFEE：基于认知控制的 LLM 特征发现引擎

提出 CoFEE（Cognitive Feature Engineering Engine），通过在 LLM 特征发现过程中施加结构化认知行为（反向链推理、子目标分解、泄漏检测回溯等）来提升特征质量。相比普通提示方法，成功率提高 15.2%，特征数量减少 29%，成本降低 53.3%。

🔗 [arXiv:2604.21584](https://arxiv.org/abs/2604.21584)

### 9. ⚖️ 生成式 AI 的公平性：用户可选的推理时间干预

提出轻量级推理时公平性框架，允许用户在文生图模型（Stable Diffusion / DALL-E）中选择公平性定义（均匀分布或 LLM 推荐的复杂策略），通过提示级干预在不修改模型的前提下调整人口统计学代表性。

🔗 [arXiv:2604.21036](https://arxiv.org/abs/2604.21036)

### 10. 📊 谁定义了"最佳"？面向交互式 LLM 排行榜评估

被 FAccT 2026 接收。深度分析 LMArena (Chatbot Arena) 数据集的偏差问题，发现主题分布严重倾斜、模型排名在不同提示切片上存在差异。提出交互式可视化界面，让用户自定义评估优先级并探索排名变化。

🔗 [arXiv:2604.21769](https://arxiv.org/abs/2604.21769)

### 11. 🎭 对齐有"幻想"问题

被 ICLR 2026 Workshop HCAIR 接收。指出现有 AI 对齐研究假设用户能清晰表达目标和需求，但行为研究表明用户在接触 AI 系统时往往目标尚未形成——这种"Fantasia 交互"可能导致表面有用但实质偏离用户需求的行为。呼吁 AI 应主动帮助用户形成和完善意图。

🔗 [arXiv:2604.21827](https://arxiv.org/abs/2604.21827)

### 12. 🧩 GS-Quant：面向知识图谱补全的粒度语义量化

被 ACL 2026 接收。提出 GS-Quant 框架，通过粒度语义增强和生成式结构重建，将知识图谱实体编码为语义一致、结构分层的离散代码，使 LLM 能以"同构于自然语言生成"的方式推理图结构。在 KGC 基准上显著超越现有方法。

🔗 [arXiv:2604.21649](https://arxiv.org/abs/2604.21649) | [GitHub](https://github.com/mikumifa/GS-Quant)
